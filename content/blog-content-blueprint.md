# AiPay Blog — Content Blueprint & SEO Playbook

**Audience:** Казахстанские владельцы малого бизнеса (Instagram-магазины, бьюти, медцентры, пекарни, рестораны, инфобизнес, рынок/розница)
**Language:** Русский (основной), Казахский (перевод)
**Platform:** Next.js App Router + MDX (`content/blog/ru/*.mdx`)
**Base URL:** `https://aipay.kz/ru/blog/`
**Blog index meta:** "Блог AiPay — Автоматизация Kaspi Pay"

---

## Часть 1: MDX frontmatter — обязательный шаблон

Каждый файл в `content/blog/ru/` начинается с этого блока:

```yaml
---
title: "Полный заголовок статьи (50–60 символов, ключевое слово ближе к началу)"
description: "Мета-описание: выгода + ключевое слово + призыв (120–155 символов)"
date: "ГГГГ-ММ-ДД"
tags: ["kaspi-pay", "тег-сегмента", "тип-контента"]
---
```

**Правила frontmatter:**

| Поле | Требование |
|---|---|
| `title` | 50–60 символов. Начинать с ключевого слова или числа. Не дублировать H1. |
| `description` | 120–155 символов. Содержит: проблему или выгоду, ключевое слово, намёк на решение. |
| `date` | ISO 8601. Свежесть контента влияет на CTR в выдаче. |
| `tags` | 2–4 тега. Первый тег — всегда якорный (напр. `kaspi-pay`). |

## Часть 2: SEO on-page checklist — обязателен для каждой статьи

### A. Технический минимум (проверяется до публикации)

- [ ] `title` (frontmatter) содержит целевое ключевое слово, 50–60 символов
- [ ] `description` (frontmatter) 120–155 символов, без дублирования заголовка
- [ ] H1 в тексте — только один, ≠ title, но синонимичен ему
- [ ] Первое ключевое слово встречается в первом абзаце (первые 100 слов)
- [ ] Ключевое слово встречается в минимум одном H2
- [ ] Плотность ключевых слов: 1–2% (1 упоминание на 50–100 слов), без форсирования
- [ ] Alt-текст для каждого изображения содержит ключевое слово или описание
- [ ] Canonical URL соответствует пути статьи (генерируется автоматически через `generateMetadata`)
- [ ] Внутренних ссылок: минимум 2, максимум 4 на статью
- [ ] Длина статьи соответствует норме типа контента (см. раздел 4)
- [ ] Ни одного абзаца длиннее 4 строк — для мобильной читабельности
- [ ] Есть таблица, список или выделенный блок в первой трети статьи
- [ ] Статья заканчивается CTA-блоком (см. раздел 6)

### B. Семантическое ядро для русскоязычного B2B

**Якорные запросы (высокий коммерческий интент):**
- `kaspi pay автоматизация`
- `как принимать оплату kaspi бизнес`
- `верификация kaspi pay`
- `kaspi pay api интеграция`
- `автоматическое подтверждение оплаты kaspi`

**Длинный хвост (low competition, хорошая конверсия):**
- `как подключить kaspi pay к instagram магазину`
- `kaspi pay webhooks для crm`
- `автоматические счета kaspi по номеру телефона`
- `предоплата при бронировании kaspi pay бьюти`
- `kaspi pay без скриншотов`
- `поддельный чек kaspi pay как проверить`
- `kaspi pay интеграция для медцентра`

**LSI-слова** (встраивать органично, не форсировать):
счёт, терминал, верификация, webhook, CRM, бот, WhatsApp, подтверждение, предоплата, статус оплаты, фискальный чек, API, автоматизация, малый бизнес Казахстан

### C. Факторы ранжирования для русскоязычного рынка (Яндекс + Google.kz)

1. **E-E-A-T сигналы:** Упоминайте реальные имена клиентов (Darmed и др.), числа, даты кейсов
2. **Yandex ИКС:** Поведенческие факторы критичны — пишите так, чтобы читали до конца
3. **Перелинковка:** Особенно важна для нового сайта (см. раздел 7)
4. **Свежесть:** Для Казахстана актуальность "Kaspi Pay 2024/2025" повышает CTR
5. **Мобильная читабельность:** Аудитория читает с телефона, абзацы ≤ 3–4 строк
6. **Скорость загрузки:** MDX рендерится на сервере (Next.js RSC) — изображения через `<img>` с `loading="lazy"`

## Часть 3: Формулы хуков для русскоязычного B2B

### Тип A — "Стоимость проблемы" (лучший для владельцев бизнеса)

> **Формула:** `[Конкретное число] + [болевое действие] + [скрытые потери] + [масштаб проблемы]`

**Пример:**
> Если у вас 30 заказов в день через Kaspi Pay, вы тратите минимум 1,5 часа только на сверку скриншотов. Это 45 часов в месяц — почти целая рабочая неделя — на то, что можно автоматизировать за один вечер.

---

### Тип B — "Переворот ожидания" (работает для how-to и кейсов)

> **Формула:** `[Общепринятое мнение] + НО + [реальность, которую вы знаете]`

**Пример:**
> Большинство думает, что Kaspi Pay — это просто QR-код на кассе. На самом деле, это полноценный платёжный API, через который можно автоматизировать выставление счетов, верификацию и отправку фискальных чеков — без единой ручной проверки.

---

### Тип C — "Сценарий в 2 часа ночи" (для сегментов с ночными продажами)

> **Формула:** `[Конкретная ситуация] + [что происходит без автоматизации] + [что могло бы быть]`

**Пример:**
> Вы запустили рекламу в Instagram. В 23:47 пришли 8 заявок. Клиенты пишут «оплатил», вы видите непрочитанные сообщения только утром. Три из них уже купили у конкурента. AiPay подтверждает оплату и запускает ваш процесс — в 23:47, без вашего участия.

---

### Тип D — "Конкретный результат клиента" (для кейсов и технических гайдов)

> **Формула:** `[Имя/тип бизнеса] + [было] + [стало] + [конкретная цифра]`

**Пример:**
> Медицинский центр Darmed экономил 2 часа ежедневно на ручном выставлении счетов и контроле оплат. После подключения AiPay весь процесс — от создания счёта до подтверждения — занимает меньше минуты.

---

### Тип E — "Вопрос-провокация" (для SEO-вопросных запросов)

> **Формула:** `[Вопрос, который задаёт аудитория] + [неочевидный ответ]`

**Пример:**
> Как проверить, что клиент действительно оплатил через Kaspi Pay, а не прислал фейковый скриншот? Ручная проверка не защищает — AiPay подтверждает статус напрямую через API, не опираясь на скриншоты.

## Часть 5: Размещение CTA — стратегия

### Принцип: CTA появляется 3 раза, каждый раз — другой формат

#### CTA 1 — Мягкий (середина статьи, после проблемы)

Размещение: сразу после секции "до автоматизации" или после описания болевой точки.
Формат: inline-ссылка или callout-блок.

```mdx
> **Хотите посчитать, сколько времени тратите на сверку оплат?**
> Попробуйте AiPay 7 дней бесплатно — [оставьте заявку](/ru/#contact).
```

**Тон:** информационный, без давления. "Если хотите — можно попробовать"

---

#### CTA 2 — Средний (после секции "результаты" или "шаг 3–4")

Размещение: после самого сильного раздела — результаты кейса или ключевого шага гайда.
Формат: выделенный блок с конкретной выгодой.

```mdx
---

### Подключите это за 1 час

AiPay берёт весь цикл: счёт → оплата → подтверждение → ваш процесс.
7 дней бесплатно, без карты.

[Начать →](/ru/#contact)

---
```

**Тон:** конкретный, ориентированный на действие, без агрессии.

---

#### CTA 3 — Финальный (последний блок статьи, жёсткий)

Размещение: конец статьи, после итогов.
Формат: уже встроен в `/ru/blog/[slug]/page.tsx` как кнопка "Попробовать AiPay бесплатно".
В MDX-контенте перед этой кнопкой добавляйте 2–3 строки контекста:

```mdx
Если ваш бизнес принимает оплаты через Kaspi Pay и вы до сих пор проверяете скриншоты вручную — это можно исправить сегодня.
Подключение занимает около 1 часа. Первые 7 дней — бесплатно.
```

**Тон:** прямой, с urgency (но без "только сейчас!").

## Часть 6: Внутренние ссылки — стратегия для 6 статей

### Принципы

1. **Каждая статья ссылается на 2–3 других страницы сайта** (не только на другие посты)
2. **Каждая статья ссылается минимум на 1 другой пост блога** (когда появится второй)
3. **Приоритет ссылок: homepage → /developers → /integrations/* → /industries/* → другие посты**
4. **Якорный текст:** описательный, с ключевым словом, не "нажмите здесь" или "читайте тут"

### Карта внутренних ссылок

Ниже — рекомендуемые связки для типичных постов. Адаптируйте под конкретный контент.

| Из этого поста | Ссылайтесь на | Якорный текст |
|---|---|---|
| How-to: Instagram | `/ru/industries/instagram-shop-kaspi-invoices` | "автоматические счета для Instagram-магазина" |
| How-to: Instagram | `/ru/integrations/pulseai` | "Kaspi Pay в WhatsApp-ботах" |
| How-to: бьюти/бронирование | `/ru/industries/beauty-salon-prepayment` | "предоплата при бронировании для салона" |

| Технический гайд: API | `/ru/developers` | "документация AiPay API" или "для разработчиков" |

| Любой пост | `/ru/blog` | "другие статьи о Kaspi Pay" (в первом посте) |

### Правила якорных текстов

- Хорошо: `[автоматизация Kaspi Pay для медцентра](/ru/industries/medical-center-payments)`
- Хорошо: `[как работают webhooks AiPay](/ru/developers)`
- Плохо: `[нажмите здесь](/ru/developers)`
- Плохо: `[читайте подробнее](/ru/blog/some-post)`
- Плохо: тот же якорный текст дважды на одной странице, ведущий на разные URL

### Стратегия для первых 6 постов (новый блог)

При 6 постах используйте "звёздную" структуру: один "столп" + 5 "сателлитов".

**Столп-пост** (самый широкий запрос, длина 2 000+ слов):
- Пример: "Полное руководство по автоматизации Kaspi Pay для малого бизнеса в Казахстане"
- Все 5 остальных постов ссылаются на него

## Часть 8: Критерии качества перед публикацией

### Контентные

- [ ] Хук — конкретный, с числами, без обобщений
- [ ] Каждый H2 добавляет ценность, а не просто "обёртка"
- [ ] Нет воды ("В современном мире...", "Как известно...", "Несомненно...")
- [ ] Тон — прямой, без канцелярита, как говорит опытный предприниматель
- [ ] Минимум 1 конкретная цифра в каждом H2-разделе
- [ ] Финальный раздел или итог — не просто пересказ, а action-ориентирован

### SEO

- [ ] Title ≠ H1, но семантически близки
- [ ] Ключевое слово в title, H1, первом абзаце, минимум одном H2
- [ ] Description — завершённое предложение, 120–155 символов
- [ ] Внутренние ссылки: 2–4, с описательными якорями
- [ ] Нет битых ссылок (проверьте каждый path)
- [ ] Нет дублирующихся title/description с другими постами

### Форматирование

- [ ] Абзацы ≤ 4 строк (для мобильного)
- [ ] Первый список или таблица — в первой трети текста
- [ ] Разделители (`---`) перед CTA-блоками
- [ ] Код-блоки с обрамлением `` ``` `` (технические гайды)
